# Practical No 1
**Title:** Assignment 1: Launching an Amazon EC2 Instance

**Objective:**
- To understand how to launch and configure a virtual machine (EC2 instance) on Amazon Web Services (AWS).

**Details / Steps to Run:**
Step 1: Sign in to the AWS Management Console, navigate to the EC2 dashboard, and click the Launch instance button.
Step 2: In the "Name and tags" section, type a name for your instance (e.g., "My Web Server").
Step 3: Scroll to "Application and OS Images (Amazon Machine Image)". Select an AMI, such as the Free tier eligible Amazon Linux 2023 AMI.
Step 4: Under "Instance type", select your desired hardware configuration. The t2.micro instance is Free tier eligible and standard for this tutorial.
Step 5: Under "Key pair (login)", click Create new key pair. Store this downloaded file in a safe place, as you will need it to securely connect to your instance later.
Step 6: In the "Network settings" / "Firewall (security groups)" section, choose to create a new security group. Check the box for Allow SSH traffic from Anywhere (0.0.0.0/0). You can optionally allow HTTP/HTTPS traffic if setting up a web server.
Step 7: Under "Configure storage", define your root volume size. The free tier allows up to 30 GB, but the default 8 GiB gp3 volume is sufficient for a basic Linux server.
Step 8: Review the "Summary" panel on the right side to ensure all configurations are correct, then click the Launch instance button.
Step 9: Once the instance state shows as "Running", select the instance ID and click Connect. You can use EC2 Instance Connect (using the default user ec2-user) or an SSH client with your key pair to access the server.

**Conclusion:** We have successfully learned how to launch, configure, and connect to an Amazon EC2 instance using the AWS Management Console.
